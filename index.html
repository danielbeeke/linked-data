Components used: Linked Data / DBpedia

Paraphrased excerpt of Wikipedia

Linked Data is structured data which is interlinked with other data so it becomes more useful through semantic queries. Part of the vision of linked data is for the Internet to become a global database.

Components used: Linked Data / DBpedia
  • DBpedia is built upon the Wikipedia info boxes
  • Maintained by a group of German universities
  • Their ontology uses words
Components used: Linked Data / DBpedia
  • Wikidata is another Wikipedia SPARQL endpoint
  • Their ontology uses codes like wd:Q5593
  • Nice interactive SPARQL editor
  • Maintained by Wikimedia
Components used: uHTML
  • A micro HTML/SVG renderer
  • Somewhat similar to React
  • Has support for async components / templates
  • No virtual DOM
  • Uses tagged template literals
Components used: uHTML

Components used: Github Pages
  • Easy way of deploying a simple site
  • Used 404.html as a fallback
  • Free to use own domain name
  • Easy to configure via CNAME
  • Deploy with a git push
Components used: SPARQL queries
  • A RDF query language
  • Schemaless in regard to tables
  • But uses ontologies (predicate) for structure
  • Queries: subject, predicate and object
Components used: SPARQL queries
  • https://dbpedia.org/sparql, an endpoint to try SPARQL queries on
  • https://yasgui.triply.cc, a nice SPARQL editor
